masalah
problem

Dilema klasik game Web3.

The classic Web3 game dilemma.

Kalau semua aksi pemain dikirim ke blockchain, game jadi lag dan gas fee mahal. Tapi kalau gameplay sepenuhnya off-chain, pemain bisa eksploitasi skor pakai skrip atau bot — menguras reward pool secara curang. Industri GameFi arcade terjebak di antara dua ekstrem ini.

Sending every player action to the blockchain makes the game laggy and expensive. But if gameplay is fully off-chain, players can exploit scores with scripts or bots — draining the reward pool unfairly. The arcade GameFi industry is stuck between these two extremes.

solusi
solution

Pisahkan gameplay dari transaksi.

Separate gameplay from transactions.

Stack Block Arcade memisahkan gameplay (off-chain, di browser) dari transaksi keuangan (on-chain). Pemain beli tiket dan klaim reward lewat smart contract, tapi gameplay-nya sendiri berjalan mulus tanpa transaksi sama sekali — sambil tetap divalidasi server untuk mencegah cheat.

Stack Block Arcade separates gameplay (off-chain, in-browser) from financial transactions (on-chain). Players buy tickets and claim rewards via smart contract, but gameplay itself runs smoothly with zero transactions — while still server-validated to prevent cheating.

how it works

Dari beli tiket sampai klaim reward.

From buying a ticket to claiming the reward.

01

Beli tiket (on-chain)

Buy ticket (on-chain)

Pemain hubungkan wallet dan bayar biaya masuk lewat fungsi buy_ticket() di smart contract.

Player connects wallet and pays entry fee via the buy_ticket() smart contract function.

02

Inisiasi sesi

Session initiation

Smart contract mengunci token dan menerbitkan SessionID unik untuk pemain.

Smart contract locks the token and issues a unique SessionID for the player.

03

Bermain (off-chain)

Play (off-chain)

Game berjalan di browser, mencatat koordinat dan timestamp tiap balok secara lokal.

Game runs in-browser, recording coordinates and timestamp for each block locally.

04

Validasi anti-cheat

Anti-cheat validation

Server mensimulasikan ulang log koordinat untuk memastikan skor wajar dan tidak hasil hack.

Server re-simulates the coordinate log to confirm the score is plausible and not hacked.

05

Klaim reward (on-chain)

Claim reward (on-chain)

Server beri signature digital, lalu smart contract verifikasi dan kirim reward ke wallet pemain.

Server issues a digital signature, then the smart contract verifies it and sends the reward to the player's wallet.
