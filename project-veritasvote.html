masalah
problem

Krisis kepercayaan di voting konvensional.

The trust crisis in conventional voting.

Bagaimana kita bisa yakin setiap suara dihitung adil? Sistem voting terpusat rentan manipulasi di balik layar, dan rentan terhadap serangan flash loan — pihak bermodal besar meminjam token sesaat hanya untuk memenangkan voting sepihak.

How can we be sure every vote is counted fairly? Centralized voting systems are vulnerable to behind-the-scenes manipulation, and to flash loan attacks — well-funded actors borrowing tokens momentarily just to swing a vote.

solusi
solution

Snapshot voting, eksekusi otomatis.

Snapshot voting, automatic execution.

VeritasVote mengunci hak suara lewat snapshot pada blok pembuatan proposal — mencegah manipulasi flash loan. Begitu proposal disetujui mayoritas, sistem mengeksekusi keputusan secara otomatis, lengkap dengan timelock pengaman.

VeritasVote locks voting power via snapshot at the block when a proposal is created — preventing flash loan manipulation. Once a proposal is approved by majority, the system executes the decision automatically, complete with a safety timelock.

alur kerja
how it works

Tiga fase siklus governance.

Three phases of the governance cycle.

01

Pembuatan proposal

Proposal creation

Anggota yang memenuhi ambang batas token mengajukan proposal; kontrak snapshot kepemilikan token di blok tersebut.

Members meeting the token threshold submit a proposal; the contract snapshots token ownership at that block.

02

Periode voting

Voting period

Pemegang token memilih For / Against / Abstain, dihitung proporsional dari saldo snapshot.

Token holders vote For / Against / Abstain, weighted proportionally from the snapshot balance.

03

Evaluasi & eksekusi

Evaluation & execution

Kontrak mengecek kuorum dan mayoritas; jika lolos, eksekusi otomatis terpicu setelah jeda timelock.

Contract checks quorum and majority; if passed, automatic execution triggers after the timelock delay.
